# Implementation Plan: CareerCraft Bug Fixes

## Overview

28 tasks covering 29 bugs across backend and frontend. Ordered by severity: Critical blockers first (app won't start), then High (features broken), then Medium (bad UX), then Low (minor issues), then property-based tests last.

Critical tasks (1–4) must be completed before any other work — the server won't start and agents won't run without them. High tasks (5–10) unblock core features. Medium and Low tasks (11–27) can be done in parallel once the critical and high fixes are in. Task 28 (PBT) depends on all prior tasks.

## Notes

- Task 4 rewrites the RAG vector store layer. After the fix, run a manual smoke test: upload a PDF resume and verify it is chunked, embedded, and retrievable.
- Task 19 adds query params to `list_applications`. Ensure the existing `status` filter param still works (regression: Requirement 3.11).
